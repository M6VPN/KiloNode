# KiloTNC KISS

KiloTNC is planned as a direct KISS hardware path for KiloNode. The current M2
work keeps this as receive-oriented documentation and configuration guidance.
Tests do not require KiloTNC hardware.

Use the serial, PTY, TCP, or Unix receive bench examples that match the transport
being tested. Keep transmit disabled unless a later lab-only milestone opens a
specific transmit gate.
