# M2.4 Windowed Loopback

M2.4 adds simulator-only AX.25 window tracking for loopback I-frame payload
flows.

## Goal

The in-memory loopback simulator can keep more than one segmented I frame in
flight when `window` is greater than 1. The sender records outstanding frame
diagnostics, the peer returns RR frames, and the sender clears acknowledged
frames without touching the real TX queue.

## Checks

Primary checks:

- `./build/test_ax25_loopback_window`
- `./build/test_ax25_loopback_segment`
- `./scripts/ax25-loopback-fixtures.sh`
- `./scripts/m2-readiness-check.sh`
- `./scripts/ax25-no-transmit-check.sh`

Fixture coverage includes window-size 1 baseline delivery, window-size 2 text
segmentation, and window-size 3 binary segmentation.

## Limitations

- Window diagnostics are loopback-only.
- No retransmission dispatch.
- No recovery for out-of-order reassembly.
- No live daemon payload segmentation.
- No shell or BBS payload binding.
- No real TX queue writes or dispatch.
- No FX.25 wrapping or FEC.
