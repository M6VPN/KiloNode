# M2.5 Retransmission Buffer Diagnostics

M2.5 adds simulator-only retransmission-buffer diagnostics for AX.25 loopback
I-frame tests.

## Goal

The in-memory loopback simulator can record outbound I-frame bytes, mark frames
for retry after a REJ diagnostic, replay the buffered bytes inside the
simulator, and account for RR acknowledgement after replay. This proves retry
bookkeeping without writing to the real TX queue.

## Checks

Primary checks:

- `./build/test_ax25_loopback_retransmit`
- `./build/test_ax25_loopback_script`
- `./scripts/ax25-loopback-fixtures.sh`
- `./scripts/m2-readiness-check.sh`
- `./scripts/ax25-no-transmit-check.sh`

Fixture coverage includes one-frame REJ replay, windowed REJ replay, and a
direct replay-buffer diagnostic case.

## Limitations

- Retransmission replay is loopback-only.
- No live daemon retransmission dispatch.
- No timer-driven I-frame retransmission in the daemon.
- No real TX queue writes or dispatch.
- No shell or BBS payload binding.
- No FX.25 wrapping or FEC.

## Next Work
