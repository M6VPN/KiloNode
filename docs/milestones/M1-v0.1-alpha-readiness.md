# M1 v0.1-alpha Readiness

v0.1-alpha means KiloNode is ready for a receive-side and diagnostics tag. It
does not mean connected-mode response TX, RF BBS, NET/ROM, forwarding, or
FX.25 are ready.

Current recommended tag status: ready for a v0.1-alpha receive/diagnostics tag
after `./scripts/m1-readiness-check.sh` and `./scripts/test.sh` pass locally.

## What Works Now

- Clean-room ISC-licensed C17 KiloNode scaffold.
- KISS frame parsing and local transport scaffolding.
- Local daemon, local node shell, control socket, and `kilonodectl`.
- KiloNode-native local BBS store, local BBS shell, users, read-state, and
  read-only BBS control queries.
- Receive-only KISS/AX.25 monitoring, heard list, RX event queues, and RF
  command ingress diagnostics.
- AX.25 connected-mode diagnostics scaffolds, including connection table, state
  machine, timers, scheduler, replay harnesses, and prepared-frame diagnostics.
- Compatibility lab fixtures for synthetic transcripts, captures, observations,
  requirements, profiles, and risk reports.
- Manual capture workspace tooling for user-provided receive-only `.capture`
  files.
- Non-transmitting safety gates that verify default configs do not enable real
  TX dispatch or prepared-to-real-TX promotion.

## Deliberately Blocked

- Live CONNECT commands.
- Real connected-mode sessions.
- RF BBS access.
- AX.25 I-frame payload delivery to shell or BBS.
- Prepared AX.25 frames promoted to the real TX queue.
- Real retransmission and TX dispatch.
- NET/ROM routing.
- BBS forwarding and FBB forwarding.
- LinBPQ command compatibility.
- FX.25 FEC encode/decode and TX wrapping.

## Diagnostics Inspection

When a daemon is running, use:

```sh
./build/kilonodectl --socket /tmp/kilonode/control.sock ax25 status
./build/kilonodectl --socket /tmp/kilonode/control.sock ax25 live
./build/kilonodectl --socket /tmp/kilonode/control.sock ax25 scheduler-smoke
./build/kilonodectl --socket /tmp/kilonode/control.sock ax25 prepared
./build/kilonodectl --socket /tmp/kilonode/control.sock ax25 prepared-bridge
./build/kilonodectl --socket /tmp/kilonode/control.sock tx gates
```

BBS and local store state can be inspected with:

```sh
./build/kilonodectl --socket /tmp/kilonode/control.sock bbs status
./build/kilonodectl --socket /tmp/kilonode/control.sock bbs messages
./build/kilonode-store --help
./build/kilonode-msg --help
./build/kilonode-user --help
```

Compatibility fixtures can be inspected through `kilonode-compat` and the
scripts listed in `M1-test-inventory.md`.

## Recommendation

The M1 set is internally consistent for a receive/diagnostics alpha if the
readiness scripts pass. The release notes must state that TX response paths,
CONNECT, NET/ROM, RF BBS, forwarding, LinBPQ command compatibility, and FX.25
FEC remain future work.
