# M2.2 I-Frame Loopback

M2.2 adds AX.25 I-frame build/decode helpers and extends the offline loopback
simulator with bounded payload delivery diagnostics.

## Scope

Implemented scope:

- modulo 8 I-frame control-byte build/decode helpers
- raw AX.25 I-frame builder
- raw AX.25 I-frame decode helper
- text and binary payload loopback fixtures
- delivery and rejected-delivery diagnostics
- RR acknowledgement after valid I frames
- REJ diagnostics after sequence mismatch
- loopback reports with I/RR and payload counters

## Checks

Run:

```text
./build/test_ax25_i_frame
./build/test_ax25_payload_delivery
./build/test_ax25_loopback_payload
./scripts/ax25-loopback-fixtures.sh
./build/kilonode-compat run-ax25-loopback tests/fixtures/ax25-loopback/connect-i-rr-disconnect.loop
```

## Safety

The work is simulator-only. It does not expose CONNECT, bind payloads to shell
or BBS code, write to the real TX queue, dispatch frames, or generate FX.25.

## Limits

Modulo 128 I frames remain planned. Live daemon I-frame delivery, live
connected-mode sessions, retransmission buffers, shell or BBS binding, and real
RF response TX remain future milestones.
