# M2.1 Loopback Simulator

M2.1 adds an offline in-memory AX.25 connected-mode loopback simulator. It runs
two KiloNode AX.25 diagnostic endpoints against each other without real RF,
KISS transports, daemon CONNECT commands, real TX queue writes, or dispatch.

## Scope

The simulator covers:

- local connect request
- SABM or SABME prepared frame generation
- peer receive of setup frame
- peer UA generation
- initiator receive of UA
- connected state establishment
- simulator-local I-frame and RR exchange
- DISC and UA disconnect
- T1 timeout retry followed by late UA

## Checks

Run:

```text
./scripts/ax25-loopback-fixtures.sh
./scripts/m2-readiness-check.sh
./build/kilonode-compat run-ax25-loopback-dir tests/fixtures/ax25-loopback
```

The committed fixtures assert:

- `tx-writes 0`
- `dispatch-calls 0`
- `fx25-frames 0`

## Limits

The simulator does not expose CONNECT to users or daemon control. It does not
promote prepared frames to the real TX queue and does not dispatch frames.

I-frame support is simulator-local for loopback proof only. Live I-frame TX,
payload delivery to shell or BBS, retransmission dispatch, and RF BBS access are
future milestones.

FX.25 remains planned. Loopback reports must keep FX.25 frame count at zero.
