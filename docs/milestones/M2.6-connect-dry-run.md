# M2.6 CONNECT Dry-Run Planner

M2.6 adds an offline AX.25 CONNECT dry-run planner for local-admin diagnostics.

## Goal

The dry-run planner validates a hypothetical local CONNECT request and reports
the planned connection state, SABM or SABME intent, prepared-to-TX bridge
status, and no-transmit counters. It does not create sessions or expose CONNECT
to daemon control, node shell, BBS shell, RF commands, or any transport.

## Checks

Primary checks:

- `./build/test_ax25_connect_dry_run`
- `./build/test_ax25_connect_dry_run_script`
- `./scripts/ax25-connect-dry-run-fixtures.sh`
- `./scripts/m2-readiness-check.sh`
- `./scripts/ax25-no-transmit-check.sh`

Fixture coverage includes modulo-8 SABM planning, modulo-128 SABME planning,
paclen/window boundary planning, invalid callsign rejection, and unsafe TX
expectation rejection.

## Limitations

- No live CONNECT command.
- No daemon control mutation.
- No connection table insertion.
- No real TX queue writes or dispatch.
- No shell or BBS binding.
- No RF BBS, NET/ROM, BBS forwarding, or FBB forwarding.
- No FX.25 wrapping or FEC.
