# M2.8 External Modem Scaffold

M2.8 adds a generic external modem status scaffold for KiloNode. It is aimed at
the hobbyist product path and keeps all external modem integration non-
transmitting.

## Implemented Scope

- `external-modem` config blocks.
- Built-in modem profiles for `kiss-tcp`, `mercury-ofdm`, `vara-hf`,
  `vara-fm`, `ardop`, and `generic-tcp`.
- Read-only control commands for modem status and profiles.
- `kilonodectl` commands for modem status and profile inspection.
- Safe example configs for external modem status and Mercury planning.
- Config and no-transmit checks for modem TX, CONNECT, and autostart flags.

## Mercury OFDM Status

Mercury OFDM is listed as a first-class planned target. KiloNode does not assume
Mercury interface details yet. Interface discovery is required before any
adapter implementation.

## Limitations

- No external modem process launch.
- No modem socket connection attempt.
- No Mercury, VARA, ARDOP, or Dire Wolf protocol implementation.
- No CONNECT.
- No real transmit.
- No shell or BBS payload binding.
- No FX.25 FEC or wrapping.
