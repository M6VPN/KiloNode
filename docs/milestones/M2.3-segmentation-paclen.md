# M2.3 Segmentation and Paclen

M2.3 adds AX.25 connected-mode payload segmentation policy and loopback
diagnostics.

## Goal

The in-memory loopback simulator can split a payload into paclen-sized I-frame
segments, move each raw AX.25 I frame to the peer endpoint, process RR
acknowledgements, and record reassembled payload diagnostics.

## Checks

Primary checks:

- `./build/test_ax25_paclen`
- `./build/test_ax25_segmenter`
- `./build/test_ax25_reassembly`
- `./build/test_ax25_loopback_segment`
- `./scripts/ax25-loopback-fixtures.sh`
- `./scripts/ax25-no-transmit-check.sh`

Fixture coverage includes segmented text, segmented binary, paclen boundary,
paclen 1 stress, and window-size 1 segmented delivery.

## Limitations

- Window-size 1 only for segmented send.
- No retransmission buffer.
- No out-of-order reassembly recovery.
- No live daemon payload segmentation.
- No shell or BBS payload binding.
- No real TX queue writes or dispatch.
- No FX.25 wrapping or FEC.

## Next Work

Future milestones can add explicit outstanding-frame tracking, controlled
windowed loopback, retransmission buffer diagnostics, and later live daemon
integration behind the existing no-transmit gates.
